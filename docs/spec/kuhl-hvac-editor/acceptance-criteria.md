### REQ-001: 建築ノード完全削除 🔵

**信頼性**: 🔵 *ユーザヒアリング: 完全削除確定*

#### Given（前提条件）
- 既存のPascal Editor V2コードベースが存在する
- 建築ノード（Wall, Slab, Ceiling等 14種）、システム（8件）、レンダラー（17件）、ツール（25+件）が実装済み

#### When（実行条件）
- Phase 0のフォーク＆削除を実行する

#### Then（期待結果）
- 建築関連のノードスキーマ、システム、レンダラー、ツールがすべて削除されている
- 共通基盤（BaseNode, useScene, event bus, sceneRegistry, dirty tracking, Zundo）が正常に動作する
- TypeScriptコンパイルエラーがない
- 既存テストが削除対象以外はパスする

#### テストケース

##### 正常系

- [ ] **TC-001-01**: 共通基盤の動作確認 🔵
  - **入力**: `useScene.getState().createNode()` で空のノードを作成
  - **期待結果**: ノードが `nodes` 辞書に追加され、`dirtyNodes` にマークされる
  - **信頼性**: 🔵 *既存useScene APIの継続使用*

- [ ] **TC-001-02**: undo/redo動作確認 🔵
  - **入力**: ノード作成後、undo → redo を実行
  - **期待結果**: ノードが削除→復元される
  - **信頼性**: 🔵 *既存Zundoの継続使用*

- [ ] **TC-001-03**: AnyNode ユニオンの再構成確認 🔵
  - **入力**: 空調ノードタイプで `AnyNode.parse()` を実行
  - **期待結果**: 全空調ノードタイプが正しくパースされる
  - **信頼性**: 🔵 *設計文書 §2.3*

##### 異常系

- [ ] **TC-001-E01**: 旧ノードタイプのパースエラー 🟡
  - **入力**: `AnyNode.parse({ type: 'wall', ... })`
  - **期待結果**: Zodバリデーションエラーが発生する
  - **信頼性**: 🟡 *削除後の整合性確認*

### REQ-002: 空調スキーマ定義 🔵

**信頼性**: 🔵 *設計文書 §2.1-2.3*

#### Given（前提条件）
- BaseNodeが定義済み
- objectId(), nodeType() ヘルパーが利用可能

#### When（実行条件）
- 各空調ノードスキーマを `.parse()` で生成する

#### Then（期待結果）
- 全ノードタイプ（21種）が正しく生成される
- IDが自動生成される（例: `zone_abc123`, `ahu_def456`）
- デフォルト値が正しく設定される

#### テストケース

##### 正常系

- [ ] **TC-002-01**: HvacZoneNodeの生成 🔵
  - **入力**: `HvacZoneNode.parse({ zoneName: '事務室A', usage: 'office', floorArea: 100 })`
  - **期待結果**: ID自動生成、designConditions のデフォルト値（夏26℃/50%RH, 冬22℃/40%RH）が設定される
  - **信頼性**: 🔵 *設計文書 §2.2 HvacZoneNode*

- [ ] **TC-002-02**: AhuNodeの生成 🔵
  - **入力**: `AhuNode.parse({ tag: 'AHU-101', equipmentName: '空調機No.1', ports: [...] })`
  - **期待結果**: ID自動生成、position/rotation/dimensionsのデフォルト値、LODデフォルト100
  - **信頼性**: 🔵 *設計文書 §2.2 AhuNode*

- [ ] **TC-002-03**: DuctSegmentNodeの生成 🔵
  - **入力**: `DuctSegmentNode.parse({ start: [0,0,0], end: [5,0,0], shape: 'rectangular', width: 400, height: 300, medium: 'supply' })`
  - **期待結果**: material デフォルト 'galvanized'、insulation デフォルト 'none'
  - **信頼性**: 🔵 *設計文書 §2.2 DuctSegmentNode*

- [ ] **TC-002-04**: PipeSegmentNodeの生成 🔵
  - **入力**: `PipeSegmentNode.parse({ start: [0,0,0], end: [3,0,0], nominalSize: '50A', material: 'sgp', medium: 'chilled_water' })`
  - **期待結果**: insulation デフォルト 'none'
  - **信頼性**: 🔵 *設計文書 §2.2 PipeSegmentNode*

- [ ] **TC-002-05**: PortDefの定義 🔵
  - **入力**: AHUのports配列にPortDef（給気口, 還気口, 冷水入口, 冷水出口）を定義
  - **期待結果**: medium, direction, position, connectedTo が正しく設定される
  - **信頼性**: 🔵 *設計文書 §2.2 PortDef*

##### 異常系

## Phase 1: ゾーニング＋負荷概算

### REQ-103: 空調負荷概算 🔵

**信頼性**: 🔵 *設計文書 §3.2 calculateZoneLoad()*

#### Given（前提条件）
- HvacZoneNodeが作成済み
- ゾーンの用途・面積・設計条件が設定済み
- 外気条件が設定済み

#### When（実行条件）
- calculateZoneLoad() を実行する

#### Then（期待結果）
- 冷却負荷(kW)、暖房負荷(kW)、顕熱比、必要風量(m³/h) が算出される
- 負荷内訳（構造体通過熱、日射熱、内部発熱、外気負荷、隙間風）が算出される

#### テストケース

##### 正常系

- [ ] **TC-103-01**: 事務室の負荷概算 🔵
  - **入力**: usage='office', floorArea=100m², 外壁あり, glazingRatio=0.3
  - **期待結果**: coolingLoad = 100 × 190 / 1000 = 19.0 kW, heatingLoad = 100 × 100 / 1000 = 10.0 kW
  - **信頼性**: 🔵 *設計文書 §3.2 原単位テーブル*

- [ ] **TC-103-02**: サーバー室の負荷概算 🔵
  - **入力**: usage='server_room', floorArea=50m², 外壁なし
  - **期待結果**: coolingLoad = 50 × 500 / 1000 = 25.0 kW, heatingLoad = 0 kW
  - **信頼性**: 🔵 *設計文書 §3.2 原単位テーブル*

- [ ] **TC-103-03**: ガラス面積補正の適用 🔵
  - **入力**: usage='office', floorArea=100m², 外壁あり, glazingRatio=0.5 (>0.4)
  - **期待結果**: coolingLoad = 100 × 190 × 1.15 / 1000 = 21.85 kW（15%増）
  - **信頼性**: 🔵 *設計文書 §3.2 coolingFactor*

- [ ] **TC-103-04**: 必要風量の算出 🔵
  - **入力**: coolingLoad = 19.0 kW
  - **期待結果**: requiredAirflow = 19.0 × 3600 / (1.2 × 1.006 × 10) ≈ 5672 m³/h
  - **信頼性**: 🔵 *設計文書 §3.2 計算式*

- [ ] **TC-103-05**: 負荷内訳の比率 🔵
  - **入力**: coolingLoad = 19.0 kW
  - **期待結果**: transmission=25%, solar=30%, internal=30%, ventilation=10%, infiltration=5%
  - **信頼性**: 🔵 *設計文書 §3.2*

##### 異常系

- [ ] **TC-103-E01**: 未定義の用途 🟡
  - **入力**: usage='unknown_type', floorArea=100m²
  - **期待結果**: デフォルト原単位（cooling:150, heating:80）を使用
  - **信頼性**: 🟡 *設計文書 §3.2 のフォールバック*

##### 境界値

- [ ] **TC-103-B01**: 面積0m²のゾーン 🟡
  - **入力**: floorArea=0
  - **期待結果**: coolingLoad=0, heatingLoad=0, requiredAirflow=0
  - **信頼性**: 🟡 *計算ロジックから推測*

- [ ] **TC-103-B02**: 極大面積のゾーン 🟡
  - **入力**: floorArea=10000m², usage='office'
  - **期待結果**: coolingLoad=1900 kW（数値オーバーフローなし）
  - **信頼性**: 🟡 *境界値テスト*
