# Kühl HVAC Editor 準備タスク（ユーザー作業）

> **仕様**: [requirements.md](requirements.md)
> **生成日**: 2026-03-23

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・設計文書・ユーザヒアリングで明確に必要と判明したタスク
- 🟡 **黄信号**: 要件定義書・設計文書から妥当に推測されるタスク
- 🔴 **赤信号**: 推測による予防的タスク（実装時に不要と判明する可能性あり）

## 必須（実装開始前に完了が必要）

以下のタスクが完了していないと、実装フェーズでブロッカーになります。

- [ ] **web-ifcライブラリの技術検証** 🔵 *設計文書 §5.1*
  - web-ifc（WASM）がWebGPU環境で正常に動作するか検証する
  - テスト用のIFCファイル（建築躯体）を用意する
  - IFC 2x3 と IFC 4 の両方で動作確認する
  - 関連要件: REQ-106, REQ-107

- [ ] **Supabase Edge FunctionsでのIFC処理の技術検証** 🟡 *ユーザヒアリング: Edge Functions*
  - IfcOpenShellまたは代替ライブラリがEdge Functionsで実行可能か検証する
  - Edge Functionsの実行時間制限・メモリ制限を確認する
  - 不可能な場合の代替案（専用サーバー、ブラウザ側のみ等）を検討する
  - 関連要件: REQ-602

- [ ] **積算連携フォーマット仕様書の入手** 🔵 *ユーザヒアリング: 仕様確定済み*
  - みつもりくん取込形式のフォーマット仕様書を社内から入手する
  - Rebroデータリンク形式のフォーマット仕様書を入手する
  - CSV出力のカラム定義を確定する
  - 関連要件: REQ-502

- [ ] **テスト用IFCファイルの準備** 🔵 *設計文書 §5.1*
  - 建築躯体のサンプルIFCファイルを用意する（小規模・中規模・大規模）
  - 複数階の建物データを含むファイルを用意する
  - 関連要件: REQ-106, NFR-003

## 推奨（実装中に用意できればOK）

実装を開始できますが、該当機能の実装前までに準備してください。

- [ ] **空調負荷原単位テーブルの確認・カスタマイズ** 🔵 *設計文書 §3.2*
  - 国交省基準ベースの原単位値（W/m²）を社内基準と照合する
  - 必要に応じて社内標準の補正係数を追加する
  - 必要になるフェーズ: Phase 1
  - 関連要件: REQ-103, REQ-104

- [ ] **LOD200の3D形状仕様の決定** 🟡 *ユーザヒアリング: LOD100+200*
  - AHU, PAC, FCU, 制気口それぞれのLOD200形状を決定する
  - プロシージャル生成のパラメータ（ファン部分の表現等）を決定する
  - 必要になるフェーズ: Phase 2
  - 関連要件: REQ-201, REQ-208

- [ ] **ダクト規格サイズテーブルの準備** 🔵 *設計文書 §3.3*
  - 矩形ダクト標準サイズ一覧（100, 150, 200, 250, 300, ...mm）を確定する
  - 円形ダクト（スパイラルダクト）標準サイズ一覧を確定する
  - 必要になるフェーズ: Phase 3
  - 関連要件: REQ-303

- [ ] **配管規格サイズテーブルの準備** 🔵 *設計文書 §3.5*
  - JIS配管サイズ一覧（15A, 20A, 25A, ...）を確定する
  - 材質別の摩擦係数テーブルを用意する
  - 必要になるフェーズ: Phase 4
  - 関連要件: REQ-402

## 確認事項（判断が必要）

実装方針に影響するため、早めの判断・確認が推奨されます。

- [ ] **リポジトリ構成の決定** 🔵 *ユーザヒアリング: 新規スコープ @kuhl/*
  - 同一リポジトリ内で `@pascal-app/*` → `@kuhl/*` に置き換えるか
  - 新しいリポジトリ（kuhl-editor）としてフォークするか
  - monorepoのpackages/配下の構成を確定する
  - 関連要件: REQ-001, REQ-002, REQ-003

- [ ] **IFC出力の実行環境の最終決定** 🟡 *技術検証の結果次第*
  - Supabase Edge Functionsでの技術検証結果を踏まえて最終決定する
  - 代替案: 専用Pythonサーバー、ブラウザ側のみ（web-ifc export）
  - 関連要件: REQ-602

- [ ] **外気条件データベースの選定** 🟡 *設計文書 §3.2*
  - 負荷計算に使用する地域別設計外気温データのソースを決定する
  - 気象庁データ、SHASE（空気調和・衛生工学会）データ等の選択肢
  - 関連要件: REQ-103
