## 実装詳細

### 1. useScene ストア定義 🔵

**信頼性**: 🔵 *既存useScene + CLAUDE.md*

Zustand store。nodes: Record<string, AnyNode>。rootNodeIds: string[]。dirtyNodes: Set<string>。

**実装ファイル**: `packages/kuhl-core/src/store/use-scene.ts`

### 2. CRUD アクション 🔵

**信頼性**: 🔵 *既存node-actions パターン*

createNode(node, parentId): ノード追加、parentId設定、親のchildren更新、markDirty。updateNode(id, partial): 部分更新、markDirty。deleteNode(id): ノード削除、子ノード連鎖削除、親のchildren更新。

**実装ファイル**: `packages/kuhl-core/src/store/actions/node-actions.ts`

### 3. Zundo temporal store 🔵

**信頼性**: 🔵 *既存Zundoパターン*

temporal middleware。max 50 snapshots。undo()/redo() 操作。差分検出→markDirty。

## 単体テスト要件

### テストケース1: createNode 🔵

**Given**: 新規AhuNode, parentId=level_xxx
**When**: createNode(node, parentId)
**Then**: nodes に追加、parentId設定、dirtyNodes にマーク

### テストケース2: updateNode 🔵

**Given**: 既存ノードID, 部分更新データ
**When**: updateNode(id, { coolingCapacity: 100 })
**Then**: ノード更新、dirtyNodes にマーク

### テストケース3: deleteNode 連鎖削除 🔵

**Given**: 親ノード（子あり）
**When**: deleteNode(parentId)
**Then**: 親と子が全て削除される

### テストケース4: undo/redo 🔵

**Given**: createNode 実行後
**When**: undo()
**Then**: ノードが削除された状態に戻る

### テストケース5: dirtyNodes markDirty/clearDirty 🔵

**Given**: markDirty(id) 実行後
**When**: clearDirty(id)
**Then**: dirtyNodes から削除

**テストファイル**: `packages/kuhl-core/src/__tests__/store/use-scene.test.ts`
