## 実装詳細

### 1. KuhlEvents 型・イベントバス 🔵

**信頼性**: 🔵 *architecture.md イベントバス拡張*

mitt emitter。NodeEvents<type, event> パターン。hvac_zone, ahu, pac, fcu, diffuser, duct_segment, pipe_segment 等のイベント。キー形式: `{nodeType}:{suffix}` (例: `ahu:click`, `zone:enter`)。

**実装ファイル**: `packages/kuhl-core/src/events/bus.ts`

### 2. sceneRegistry 🔵

**信頼性**: 🔵 *architecture.md sceneRegistry拡張*

nodes: Map<string, THREE.Object3D>。byType: 全HVACノードタイプのSet<string>（plant, building, level, hvac_zone, ahu, pac, ... 全24種）。

**実装ファイル**: `packages/kuhl-core/src/hooks/scene-registry/scene-registry.ts`

### 3. useRegistry / useNodeEvents フック 🟡

**信頼性**: 🟡 *既存パターンから推測（@kuhl/viewer側）*

useRegistry(nodeId, type, ref): マウント時にsceneRegistryに登録、アンマウント時に解除。useNodeEvents(node, type): イベントハンドラ(onClick, onPointerEnter等)を返す。

**実装ファイル**: `packages/kuhl-viewer/src/hooks/use-node-events.ts`

## 単体テスト要件

### テストケース1: イベント発火・購読・解除 🔵

**Given**: bus.on('ahu:click', handler) 登録
**When**: bus.emit('ahu:click', event)
**Then**: handler が呼ばれる

### テストケース2: ノードタイプ別イベントキー 🔵

**Given**: 各HVACノードタイプ
**When**: イベント発火
**Then**: 正しいキー形式でイベントが配信される

### テストケース3: sceneRegistry.nodes Map 🔵

**Given**: 空のregistry
**When**: nodes.set('ahu_001', obj3d)
**Then**: nodes.get('ahu_001') で取得できる

### テストケース4: sceneRegistry.byType Set 🔵

**Given**: 空のregistry
**When**: byType.ahu.add('ahu_001')
**Then**: byType.ahu.has('ahu_001') === true

### テストケース5: useRegistry 登録・解除 🟡

**信頼性**: 🟡 *R3F環境でのテストは推測*

**Given**: useRegistry(nodeId, 'ahu', ref) 呼び出し
**When**: コンポーネントアンマウント
**Then**: registry から削除される

**テストファイル**: `packages/kuhl-core/src/__tests__/events/bus.test.ts`, `packages/kuhl-core/src/__tests__/hooks/scene-registry.test.ts`
